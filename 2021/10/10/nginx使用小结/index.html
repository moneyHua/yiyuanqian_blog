nginx使用小结

2021/10/10 | 服务器 | 阅读 | 933 字 | 3 分钟

简介

nginx是一款高性能、轻量级的Http和反向代理web服务器软件。

实际应用

  • 正向代理：代理服务器代替客户端对目标服务器进行请求并返回请求内容，如下图所示。

  • 反向代理：客户端请求对外暴露的代理服务器，代理服务器选择目标服务器进行请求转发；目标服务器返回请求结果给代理服务器，代理服务器转发请求结果给客户端，如下图所示。

  • 负载均衡：负载均衡实现方式有很多种，可通过nginx反向代理功能结合自带的均衡规则实现应用层的负载均衡。nginx负载均衡规则有如下几种：
    • 轮询：默认方式；
    • weight：权重方式；
    • ip_hash：依据ip分配方式；
    • least_conn：最少连接方式；
    • fair：响应时间方式（第三方）；
    • url_hash：依据URL分配方式（第三方）。

  • 动静分离：代理服务器将静态请求、动态请求分别转发到静态资源服务器、动态资源服务器并返回请求响应，从而提高服务器访问效率。

  • 常用超时配置说明：

超时类型 默认值 上下文 备注
client_header_timeout 60s http server 指定等待client发送一个请求头的超时时间（例如：GET / HTTP/1.1）
client_body_timeout 60s http server location 设置请求体的读超时时间
keepalive_timeout 75s http server location 指定了与client的keep-alived的连接超时时间
lingering_timeout 5s http server location 在关闭连接前，会检测是否有用户发送的数据到达服务器，如果超过lingering_timeout时间后还没有数据可读，就直接关闭连接；否则，必须在读取完连接缓冲区上的数据并丢弃掉后才会关闭连接。
proxy_connect_timeout 60s http server location 设置upstream server的连接超时时间
proxy_read_timeout 60s http server location 该指令设置与代理服务器的读超时时间。它决定了nginx会等待多长时间来获得请求的响应。这个时间不是获得整个response的时间，而是两次reading操作的时间。
proxy_send_timeout 60s http server location 这个指定设置了发送请求给upstream服务器的超时时间。超时设置不是为了整个发送期间，而是在两次write操作期间。如果超时后，upstream没有收到新的数据，nginx会关闭连接。
proxy_upstream_fail_timeout（fail_timeout） 10s upstream Upstream模块下server指令的参数，设置了某一个upstream后端失败了指定次数（max_fails）后，该后端不可操作的时间，默认为10秒。
